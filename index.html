sass基础用法二

上一节的sass基础用法一主要是从sass文件后缀名、嵌套、变量、编译等进行了介绍。这篇主要介绍mixin、继承、%点位符、函数等。
建议大家对文章中所有举的例子自己都能在sassmeister上编译一下，加深理解。

sass-mixin(混合宏)

sass中使用@mixin声明混合，可以传递参数，参数名以$符号开始，多个参数以逗号分开，也可以给参数设置默认值。声明的@mixin通过@include来调用。

一，mixin参数类型

1，无参数的mixin


@mixin title-style {
    margin: 0 0 20px 0;
    font-size: 20px;
    font-weight: bold;
    text-transform: uppercase;
}
//调用
.test{
    @include title-style;
}

2，一个参数的mixin

@mixin title-style($color) {
    margin: 0 0 20px 0;
    font-size: 20px;
    font-weight: bold;
    text-transform: uppercase;
    color: $color;
}
//调用
.test{
    @include title-style(#ccc);
}

3，多参数的mixin

@mixin title-style($color, $background) {
    margin: 0 0 20px 0;
    font-size: 20px;
    font-weight: bold;
    text-transform: uppercase;
    color: $color;
    background: $background;
}
//调用
.test{
    @include title-style(#ccc, #f50);
}

4，默认的参数

当你使用mixin的参数时，定义一个经常使用的值作为默认参数是很方便的，你只需要简单的调用mixin，不用传递参数，如果有默认的值，会把这个值传递过去。 
@mixin title-style($color, $background: #eee) {
    margin: 0 0 20px 0;
    font-size: 20px;
    font-weight: bold;
    text-transform: uppercase;
    color: $color;
    background: $background;
}
//调用
.test{
    @include title-style(#ccc);
}
//或是你可以想覆盖，传你需要的值
.test{
    @include title-style(#ccc,#000);
}
当你有多个默认参数在mixin中定义，你可以有选择的覆盖默认参数，而不需要全部重新定义。 
@mixin title-style($color: blue, $background: green) {
    margin: 0 0 20px 0;
    font-size: 20px;
    font-weight: bold;
    text-transform: uppercase;
    color: $color;
    background: $background;
}
//调用
.test{
    @include title-style($background:pink);
}

@content

@content可以用来解决css3的@media带来的问题。它可以使@mixin接受一整块样式，接受的样式从@content开始。 
@mixin header{
    .test{
        @content;
    }
}
@include header{
    width:1000px;
    height:200px;
    .logo{
        width:200px;
    }
}

二，mixin实例展示

解决浏览器前缀-单一参数

@mixin rounded($radius) {
    -webkit-border-radius: $radius;
       -moz-border-radius: $radius;
            border-radius: $radius;
}
//调用
.test{
    @include rounded(10px);
}

解决浏览器前缀-多个参数

@mixin shadow($x, $y, $blur, $color) {
    -webkit-box-shadow: $x $y $blur $color;
       -moz-box-shadow: $x $y $blur $color;
            box-shadow: $x $y $blur $color;
}
//调用
.test{
    @include shadow(0, 1px, 2px, rgba(0,0,0,.5));
}

//box-shadow设置很多个值 ，传递参数的后面加上...
@mixin box-shadow($shadow...){
    -webkit-box-shadow:$shadow;
    -moz-box-shadow:$shadow;
    box-shadow:$shadow;
}
//调用
.test{
    @include box-shadow(0 0 5px rgba(0,0,0,.3),inset 0 0 3px rgba(255,255,255,.5));
}

定义media-queries的最小最大宽度

//选择器里面调用，强调以选择器为主
@mixin screen($res-min, $res-max){
  @media screen and ( min-width: $res-min ) and ( max-width: $res-max ){
    @content;
  }
}
#header{
    @include screen(780px,1000px){
        color:red;
    }
}

//直接调用，强调以屏幕大小为主，可以方便组织所有在这个条件中的都写在一起
//如果做响应式布局，我们建议使用第二种方法，以断点为主来写样式，把某个断点下的样式全部写在一起
@include screen(780px,1000px){
    body{
        font-size:14px;
    }
}

sass基础用法一

在程序员眼中，CSS是一件很麻烦的东西，它不像js那样，CSS没有变量，没有条件语句，只是一行行单纯的描述，写起来比较费事。
书写CSS难以解决的问题：

  • 语法不够强大，比如无法嵌套书写导致模块化开发中需要书写很多重复的选择器；
  • 没有变量和合理的样式复用机制，使得逻辑上相关的属性值必须以字面量的形式重复输出，导致难以维护

为了让CSS更加简洁，适应性更强等，就给CSS加入了编程元素，叫做CSS预处理。
CSS预处理器的主要目标：提供CSS缺失的样式层复用机制，减少冗余代码，提高样式代码的可维护性。
今天介绍其中的一种css预处理器——sass

sass文件后缀名

好多人不知道sass与scss的关系是什么，明明每天说的是sass为什么文件的后缀名都是scss呢？其实scss是sass的新语法。

Sass有两种后缀名文件：

  • Sass文件是缩进式的写法，对格式要求比较严谨，末尾不能有分号，不使用大括号。Sass的老语法规则
  • Scss后缀名就是和css文件格式差不多，要使用大括号和分号，方便阅读，不容易出错。 Sass的新语法规则

sass嵌套

Sass最直观的与css的不同，就体现在视觉上的嵌套
Sass中的嵌套有两种：

这两种嵌套的目的都是一样的，减少代码量，增强代码的可读性

1，选择器嵌套是指从一个选择器中嵌套子选择器，来实现选择器的继承关系。

.header{
    .logo{
        display: block;
    }
    .list{
        margin: 10px;
    }
}

2，属性嵌套是指将带有相同前缀单词的属性提出来，作为一个公有的属性，嵌套进其他属性。(不推荐)



h3{
    font:{
        size:26px;
        weight:normal;
        family:arial;
    }
    text:{
        transform: uppercase;
        decoration: underline;
    }
}


说到嵌套，有一个不得不说的就是连体符，要给一个元素赋予样式，就必须先取到这个元素，那就需要选择器，选择器是有很多种的，比如后代选择器，兄弟选择器等，如何在编写代码的时候，少写选择器，就能实现他们之间的依赖关系？这就需要与连体符结合。

flex布局

一个可以让你告别浮动，完美实现垂直水平居中的新特性–Flex布局
Flex是Flexible Box的缩写，意为“弹性布局”旨在提供一个更加有效的方式制定，调整和分布一个容器里的项目布局，即使他们的大小是未知或者是动态的。
Flex布局主要思想是让容器有能力让其子项目能够改变其宽度，高度（甚至顺序以）以最佳方式填充可用空间。（主要是为了适应所有类型的显示设备和屏幕大小）。Flex容器会使子项目（伸缩项目）扩展来填满可用空间，或缩小他们以防止溢出容器。

基本概念

采用Flex布局的元素，称为Flex容器，简称“容器”，它的所有子元素自动成为容器成员，称为flex项目，简称“项目”
常规布局是基于块和内联流方向，而flex布局是基于flex-flow流

容器默认存在两根轴：水平的主轴（main axis）和垂直的交叉轴（cross axis）主轴的开始位置叫做main start,结束的位置叫做main end
项目在主轴方向的宽度和高度就是是主轴的尺寸。main size
项目在交叉轴方向的宽度或高度就是项目的交叉轴长度。corss size

flex-direction设置决定了主轴方向，侧轴就是主轴相对应的交叉轴

伸缩容器的属性

flex-driection

这个主要是用来创建主轴，从而定义了伸缩容器的方向

flex-wrap

这个主要用来定义伸缩容器里是单行还是多行显示,交叉轴的方向决定了新行堆放的方向

inline-block

一直想写一结篇关于inline-block的总结,但一直不知道如何下笔可以写得更全面，具体一点。而且网上相关文章也可以一搜一大堆，不知写出来的还有没有必要，但最终还是觉得被inline-block坑过的人太多了，还是记录下来，留下一点纪念好一点。

到底什么是 inline-block

inline-block 后的元素创建了一个行级的块容器，该元素内部（内容）被格式化成一个块元素，同时元素本身则被格式化成一个行内元素。

直白一点：inline-block 的元素既具有 block 元素可以设置宽高的特性，同时又具有 inline 元素默认不换行的特性。

一句话：inline-block 后的元素就是一个格式化为行内元素的块容器( Block container )

ie6,7真得不支持display:inline-block吗？

ie6中inline元素和block元素的表现：

  • 1）inline元素display:inline-block

ie6中inline元素只要触发了haslayout期表现就类似于inline-block,这里设置display:inline-block;或者zoom:1;等其他属性值可以触发haslayout,表现出来是一样的。
