React Native之css

发表于 2016-09-06   |   分类于 移动开发   |     |   浏览 次

组件是React Native应用的基石，一个典型的Rn应用的用户界面，首先是声明组件，然后组件被转换为对应的原生UI组件。定义组件外观的就是css。现在Rn的教程还不是很多，关于Rn css的介绍就更少之又少了.前段做一个rn项目，所以把自己总结的点分享一下，希望能对大家有所帮助。(文章中的React Native用Rn代替)


在Rn，你并不需要学习什么特殊的语法来定义。我们仍然是使用javaScript来写样式。所有的核心组件都接受名为style的属性，这些样式名基本上是遵循了web上的css的命令，只是按照js的语法要求使用了驼峰命名法。例：margin-top改为marginTop。


style属性可以是一个普通的javaScript对象，你还可以传入一个数组，在数组中位置后面的样式对象比前面的优先组高。




Rn css 与 web css的差别



首先大家知道，Rn并没有实现完整的css，基本上只是实现了一个css的子集，并且属性名不完全一致。


我总结了几点主要的差别：



  • 布局方式（flexbox）

  • 组件顺序（无zIndex）

  • 样式继承限制

  • 选择器

  • 样式属性与值


1）布局方式（flexbox）

在web css中，css有多种布局方式 ，而在Rn开发中，只有一种布局方式，flexbox布局。这种布局方式真心很赞，我后面会重点介绍一下。


2）组件顺序（无zIndex）

组件的层叠顺序，在web css里面有一个z-index的属性，所以无论你的结构怎么写，只要当前z-index的值够大，符合条件，层级就会比较高，节点就会显示在顶层。而在Rn的css中没有zIndex这个属性，所以层级不能通过这个值去调整。


在Rn里面，所以的组件默认都在同一个层级，顺序会根据页面里定义组件的顺序实现，节点后面的会覆盖节点前面的，内层元素覆盖外层元素。想把一个元素置顶显示，那就把他的节点元素放在页面最后。这一点，没有web css灵活。


3）样式继承限制

在web上，要想指定整个文档的所有文本得是14像素，只要设置全局的样式就可以。




当浏览器尝试渲染一个文本节点的时候，它会在树中一路向上查询，直到根节点，来找到一个具备font-size属性的元素。这个设计一个不好的地方在于任何节点都可能会有font-size属性，包括div标签。在web css中，继承是一个非常自然的行为。


在Rn中，你不能设置一整颗子树的默认的样式。想使用一个一致的文本，除非你把他封装成一个文本组件，然后在你app中反复用这个组件就可以。


注：在rn中唯一的样式继承，出现在文本标签的嵌套中




Rn组件在概念上被设计为强隔离性的：你应当可以在应用的任何位置放置一个组件，而且只要属性相同，其外观和表现都将完全相同。如果能够继承外面的样式属性，将会打破这种隔离性。 各个组件都不会被他外层的样式所污染。


选择器

没有选择器的概念，在rn中，style只是一个普通的javaScript对象，为组件添加样式的一个标识。所以在定义样式的时候，就没有在web css中的那一套基本选择器，属性选择器，伪类选择器。生活的幸福指数真是降了n个台阶了。比如我想，选中a的时候，a,下面的icon和text都要有变化，我只能分别给icon 和 text都追加的class来控制他们各自的变化，而不能只给a加,去控制icon和text的变化。很麻烦，js也需要多判断。这一点多带来不少工作量。


样式属性与值

缺失了一些样式属性:background-image,text-overflow,等


margin和padding的值的设置，只能设置一个参数。


大家在做Rn项目的，多注意，多发现，有些比较细节的就不一一列举了。


flexbox布局


我们在Rn中使用flexbox规则来指定某个组件的子元素的布局。Flexbox可以在不同屏幕尺寸上提供一致的布局结构。因为Rn只有一种布局方式，从大到小都要用flexbox布局，每一个view其实都有干布局的事。所以我们必须要彻底搞定这种布局，才能顺利的开始项目。


Rn的flexbox布局要要比web版的flexbox布局的属性要少，相对简单一点，但原理基本一样。由于我前面有一篇文章对web的flexbox进行个详细的介绍。所以这里只说一下重点，和差别的部分。



flexDirection

指定flexDirection可以决定主轴的方向。子元素是应该沿着水平轴(row)方向排列，还是沿着竖直轴(column)方向排列。


Rn的flexDirection只有两种布局方式。1）row 横向布局 2）column纵向布局 而且默认值是(column)。alignItems的默认值是stretch而不是flex-start，以及flex只能指定一个数字值


我自己画了一副图，可以很清楚的看出在不同布局的时候，justifyContent,与alignItems代表的不同方向的对齐。



justifyContent永远定义主轴方向的对齐方式，至于主轴是水平还是垂直的，就由flexDirection决定。


文本元素Text


一个用于显示文本的React组件，并且它也支持嵌套、样式，以及触摸处理
Text元素在布局上不同于其它组件：在Text内部的元素不再使用flexbox布局，而是采用文本布局。这意味着Text内部的元素不再是一个个矩形，而可能会在行末进行折叠。文本内容必须放在Text里面。


1)包裹在View元素中的Text表现为block，可以设置margin和padding的各种属性；


2)包裹在Text元素中的Text表现为inline元素，不能设置其marginTop和marginBottom， padding等用于block元素的属性


我觉得Text最好还是只设置文本的属性吧，因为设置border什么的，在安卓手机上会有问题，显示不出来，所以如果设置边距，border等，最好在Text外层，在嵌套一层View，在View上进行设置。


总结



  • Rn宽度基于pt为单位,可以通过Dimensions 来获取宽高，PixelRatio获取密度

  • flexDirection的默认值 是column

  • 文字必须放在Text元素里边，要不会报错。

  • 像Text和Image这样的基础组件无需包装在View里也可以正常显示，但这并不是提倡的，将它们包装在View中，才能更好控制其样式和布局。

  • 定位，父元素不用设置position，默认相对于父容器进行位移，且没有zIndex属性

  • 图片的显示需要指定宽度和高度,才能显示。

  • 元素超过父元素，是不可以滚动的，必须在外部套一个才可以。

  • borderTopLeftRadius这类型指定各个角的样式，在安卓不好使


注：组件能够撑满剩余空间的前提是其父容器的尺寸不为零，如果父容器既没有固定width和height,也没有设定flex,则父容器的尺寸为零，其子组件那怕使用了flex也是无法显示的。


Rn css在ios里实现的已经比较好了，在安卓中还是会有一些想不到的问题，所以在做的时候，多测试，多总结。
