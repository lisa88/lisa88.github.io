git基本介绍

发表于 2016-03-02   |   分类于 git   |     |   浏览 次

git是分布式版本控制系统。

版本控制是指在开发的过程中，能自动记录每次不同的文件改动，确保由不同人所编辑的同一档案都得到更新。


版本控制的作用是追踪文件的变化，为什么需要版本控制系统？就是当你出错了，可以很容易地找回没出错时的状态。


分布式版本控制就是每个人都可以创建一个独立的代码仓库用于管理，各种版本控制的操作都可以在本地完成。每个人修改的代码都可以推送合并到另外一个代码仓库中。



git vs svn


1）分布式与集中式


集中式

集中式版本控制系统，不但速度慢，而且必须联网才能使用。


集中式版本控制，版本库集中存放在中央服务器的，而干活的时候，用的都是自己的电脑，所以要先从中央服务器取得最新的版本，然后开始干活，干完活，再把自己的活推送给中央服务器。


分布式

分布式版本控制的每个节点都是完整仓库。


根本没有“中央服务器”，每个人的电脑上都是一个完整的版本库，这样，你工作的时候，就不需要联网了，因为版本库就在你自己的电脑上。既然每个人电脑上都有一个完整的版本库，那多个人如何协作呢？比方说你在自己电脑上改了文件A，你的同事也在他的电脑上改了文件A，这时，你们俩之间只需把各自的修改推送给对方，就可以互相看到对方的修改了。


和集中式版本控制系统相比，分布式版本控制系统的安全性要高很多，因为每个人电脑里都有完整的版本库，某一个人的电脑坏掉了不要紧，随便从其他人那里复制一个就可以了。而集中式版本控制系统的中央服务器要是出了问题，所有人都没法干活了。


2）存储方式


存储方式：用hash值来存储文件的，一次commit生成的SHA-1值对应着多个文件hash值，能迅速的找到这个文件。Git中文件的存储是怎么进行的，文件系统可以看做是一个对象，这个对象的存储着很多hash值，而这些不同hash值指向了不同的地址，这些地址上才是存储文件的实物，而且这种查找是非常快的，这个时候比如我现在三个文件，这就是一次快照，快照实则是一个对象，会有对应的一个SHA1值，之歌SHA1值指向一次commit，这次commit又会记录当前的所有文件的状态，所对应的hash值，就可以非常精确的找到这次commit，所指向的对应的文件，就可以实现随时对于版本的回退切换，以及存储一个完整的版本。


3）工作方式


git记录当时的一个文件状态的快照，svn是记录一个相应时间的文件快照。


4）分支特性


svn是一个实体分支,而git更像是个HEAD指针.git是没有严格意义上的分支概念，git的分支更像是一个指针。有一个叫HEAD的指针将指向现在的工作区的一次快照，其实就是将现在的工作区进行一次快照并记录下来，HEAD指针将决定你现在工作的区域，而分支间的切换就变成了是在切换HEAD指针。指向不同的快照，然后可以直接进行其他分支的开发，这是与svn这种物理拷贝分支的概念完全不同的。这种会让分支切换和分支创建都非常快。


5）内容完整性


git的内容完整性要优于svn。git的内容存储使用的是SHA-1哈希算法，这能确保代码内容的完整性，确保在遇到磁盘故障和网络问题时降低对版本库的破坏。


6）操作本地化


由于git有一个本地的代码仓库，所以提交不需要实时联网。svn需要连接服务器进行。


git常用命令



你本地的仓库由git的三棵“树”组成：



  • 工作区：持有实际文件的工作目录

  • 暂存区：像个缓存区域，临时保存你的改动。

  • 本地仓库：版本库，工作区有一个隐藏目录.git ,这个不算工作区，而是git版本库


git clone


从远程主机克隆一个版本库，该命令会在本地主机生成一个目录，与远程主机的版本库同名。


git clone <远程仓库地址>

git checkout


检出，创建分支和切换分支。


git checkout <分支名> 切换到该分支
git checkout -b <分支名> 新建一个分支，并切换到该分支

git add


git add把文件添加进去，实际上就是把文件修改添加到暂存区


git add

git commit


git commit提交更改，实际上就是把暂存区的所有内容提交到当前分支，本地仓库。


git commmit -m”修改备注”

git status


要随时掌握工作区的状态，使用git status命令


如果git status告诉你有文件被修改过，用git diff可以查看修改内容


git push


将文件从本地仓库推送到远程仓库


git pull


将文件更新内容从远程仓库直接拉取到工作区
