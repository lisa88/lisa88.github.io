css选择器—基本选择器

发表于 2015-11-02   |   分类于 CSS

要使用css对HTML页面中的元素实现一对一，一对多或者多对一的控制，这就需要用到CSS选择器。
HTML页面中的元素就是通过CSS选择器进行控制的。

基本选择器

1, 通配符选择器（*）

通配符选择器是用来选择所有元素，也可以选择某个元素下的所有元素

2, 元素选择器（E）

3, 类选择器（.className）

类选择器是以一独立于文档元素的方式来指定样式，使用类选择器之前需要在html元素上定义类名

类选择器还可以具备多类名(ie6不支持多类选择器)

4, id选择器（#id）

使用ID选择器之前也需要先在html文档中加注ID名称，这样在样式选择器中才能找到相对应的元素，不同的是ID选择器是一个页面中唯一的值

id选择器有几个地方需要特别注意

  • 一个文档中一个id选择器只充许使用一次，因为id在页面中是唯一的
  • id选择器不能像类选择器一样多个合并使用，一个元素只能命名一个id名
  • 可以在不同的文档中使用相同的id名

5,后代选择器（E F）

后代选择器也被称作包含选择器,所起作用就是可以选择某元素的后代元素
E F，前面E为祖先元素，Ｆ为后代元素，不论Ｆ在Ｅ中有多少层关系，都将被选中

选中demo中的所有的li元素

6, 子元素选择器（E>F）

子元素选择器只能选择某元素的子元素，其中E为父元素，而F为子元素，其中E>F所表示的是选择了Ｅ元素下的所有子元素Ｆ

选择ul下的所有的子元素li，孙元素是不选择的
ie6不支持子元素选择器

7, 相邻兄弟元素选择器（E+F）

相邻兄弟选择器可以选择紧接在另一元素后的元素，而且他们具有一个相同的父元素，

只选择h1后的第一个p元素
ie6不支持相邻兄弟选择器

8, 通用兄弟选择器（E~F）

这种选择器将选择某元素后面的所有兄弟元素，他们也和相邻兄弟元素类似，需要在同一个父元素之中，换句话说，E和F元素是属于同一父元素之内，并且F元素在Ｅ元素之后，那么E ~ F 选择器将选择中所有Ｅ元素后面的Ｆ元素。

选择h1后面的所有p元素

相邻兄弟选择器仅选择与其相邻的后面的元素（选中的仅一个元素）；而通用兄弟元素选择器，选中的是元素相邻的后面的所有兄弟元素，
ie6不支持相邻兄弟选择器

9, 群组选择器（E~F）

群组选择器是将具有相同样式的元素分组在一起，每个选择器之间使用逗号
